# MeetFlow 视频会议助手

MeetFlow 是一款轻量级视频会议助手 Web 应用，围绕办公视频会议中的会前准备、会中记录、会后纪要生成和待办事项跟进场景展开，帮助用户降低会议整理成本，提升会议协作效率。

## 一、项目背景

在日常办公场景中，视频会议经常存在以下问题：

- 会议信息分散，用户需要在不同平台中查找会议时间、参会人和会议资料；
- 会中记录成本高，参会者需要一边听会一边手动整理重点内容；
- 会后纪要整理耗时，会议结束后还需要重新梳理讨论内容、结论和风险问题；
- 待办事项缺少统一跟进，容易出现负责人、截止时间和任务状态不清晰的问题。

因此，本项目设计了一个轻量级视频会议助手，聚焦会议前后最核心的信息整理与协作流程。

## 二、核心功能

### 1. 会议工作台

- 展示今日会议数量
- 展示待生成纪要数量
- 展示未完成待办数量
- 展示已归档纪要数量
- 集中管理今日会议和待办提醒

### 2. 新建会议

- 支持创建新的会议
- 可填写会议标题、会议时间、参会人员、会议类型、会议背景和会议议题
- 创建后的会议会显示在工作台会议列表中

### 3. 会议详情

- 展示会议基本信息
- 展示会议背景和会议议题
- 支持输入或粘贴会议记录文本
- 支持进入纪要生成流程

### 4. 会议纪要生成

系统可以根据会议记录生成结构化会议纪要，包括：

- 会议摘要
- 讨论要点
- 关键结论
- 待办事项
- 风险与问题
- 下一步计划

当前版本采用本地模拟 AI 生成逻辑，保证演示稳定性，后续可扩展为真实大模型接口。

### 5. 待办事项管理

- 从会议纪要中提取待办事项
- 展示任务内容、负责人、截止时间、优先级、任务状态和来源会议
- 支持修改任务状态
- 支持刷新后保留任务数据

### 6. 纪要复制与导出

- 支持一键复制会议纪要
- 支持导出 Markdown 格式会议纪要
- 便于会后分享、归档和二次编辑

### 7. 本地数据保存

- 使用 localStorage 保存会议和待办数据
- 刷新页面后，新建会议和任务状态仍然保留

## 三、技术栈

- React
- TypeScript
- Vite
- Tailwind CSS
- lucide-react
- localStorage

## 四、产品亮点

### 1. 覆盖会议完整流程

MeetFlow 不只是一个会议纪要生成器，而是围绕会前、会中和会后形成完整闭环：

会前准备 → 会中记录 → 会后纪要 → 待办跟进

### 2. 结构化会议整理

系统将原始会议记录整理为摘要、讨论要点、关键结论、风险问题和下一步计划，提升会后信息整理效率。

### 3. 从纪要到任务的闭环

会议纪要中的待办事项可以直接进入任务管理页面，支持负责人、截止时间、优先级和状态管理。

### 4. 轻量级、可快速部署

项目采用纯前端实现，无需复杂后端服务，适合快速部署和在线演示。
